# Mobile-friendly responsive layout

## Overview

Improve the admin UI so it is **usable on small viewports**: navigation, tables, forms, and key flows should adapt without horizontal breakage or unusable tap targets. This is explicitly **after** the first MVP slice in [PLAN.md](../../PLAN.md).

- **In scope:** Responsive layout and touch-friendly patterns for existing admin surfaces.
- **Out of scope:** Native apps or offline PWA scope unless added as a separate project.

## Definition of done

- Core flows (auth, collections list/editor, records list/edit) are **usable on a typical phone width** without critical layout overflow or inaccessible controls.
- PLAN.md “Mobile-friendly responsive layout” can be checked off.
- Any new breakpoints/components follow existing Tailwind/shadcn patterns in the repo.
